Data Science consists of a certain set of tasks that convert raw data to human readable information. The data science pipeline consists of data curation, parsing, and management; exploratory data analysis; hypothesis testing and machine learning to provide analysis. In this tutorial, we will take you through the five major data steps every data scientist must do for a successful project.

 

Here are the following steps:

 

  1. We will find and collect data.
  2. We will process the data, tidy the data, and deal with missing data.
  3. We will do exploratory analysis & data vizualization.
  4. We will perform analysis, hypothesis testing, and machine learning.
  5. We will gain insight & discuss policy decision.

Step 1: Find and Collect Data

 

Each project should have an end goal. You should be solving some sort of problem or looking for some sort of intuition about this data. Finding relevant data is the first implied step.

 

What data will we need?

We will need data that is from a reliable and accurate source. Incorrect data is a data scientist's nightmare. We are going to be making judgements and predictions based on the data we collect, so we must make sure the data is authentic and accurate. Most importantly, you need data that is revelant to the problem you are trying to solve.

 

Where will we find it?

There are many reputable sources out there, but for the purposes of this tutorial we will be using data from the University of California, Irvine's websute. An overview of the data we will be using can be found here. Institutes of higher education along with government websites tend to have the most reliable and accurate data. 

 

How will we collect it?

We will be using Jupyter's i-Python-Notebook-file system, using the Python programming language, Python Data Analysis Library (Pandas), a fundamental package for scientific computing with Python called NumPy, and any other scientific libraries that we can import to help process and analyze the data. 

 

 

The following code can be used to pull this data off the UCI website and put it into a Pandas DataFrame.

We chose a set that combines socio-economic data from the US census, law enforcement data, and crime data from the FBI. We found this dataset particularly interesting because, as we will show you later, you can predict crime rates based on specific features. A priori, there is no reason to assume this data is linear or any one given feature can have significant prediction power.

Notes about the data we selected:

  • Data is described below based on original values. All numeric data was normalized into the decimal range 0.00-1.00 using an unsupervised, equal-interval binning method.
  • Attributes retain their distribution and skew (e.g. the population attribute has a mean value of 0.06 because most communities are small). I.e. An attribute described as 'mean people per household' is actually the normalized (0-1) version of that value.

Step 2: Data Processing

 

The normalization preserves rough ratios of values within an attribute (e.g. double the value for double the population within the available precision - except for extreme values (all values more than 3 SD above the mean are normalized to 1.00; all values more than 3 SD below the mean are normalized to 0.00)).  

Part 1: Tidying data

When data is sparse, e.g. there are a plethora of missing values, we want to organize the data is such a way as to make it more dense. This is called melting the data. For this project, we will melt the data later in order to make a prediction about this data set using a machine learning algorithm. For now, we will sort the values of Violent Crimes Per Community, population of the comunity, and household size. This will sort the data by communities by the most violent crime, population, and household size at the top of the data frame. We can sort the data in ascending or descending order.


Part 2. Missing Data

This dataframe initially contains 1994 rows. First, we drop the columns that are not predictive: state, county, community, and fold. We leave in communityname for now to make this easier to map, though it will not be used for prediction. If we drop every column with more than 200 ?'s, which represents missing data from the dataframe, we end up with 102 columns. We will now impute the data, which means we would add a represented/typical value where missing points occur.





Next, we want to iterate through each column, besides the communityName since its the only attribute that is a string (we will leave it in our dataframe for now). During each iteration, we will convert each cell in the dataframe that contains a missing value ("?") with NumPy's NaN object. This represents an object that is 'Not A Number'. We can easily drop the columns that have 200 or more missing data entries using Pandas' to_numeric function with the 'coerce' parameter, which will convert all the ?'s to NaN's.

In [3]:
for col in data.drop('communityname',1):
    data[col]= pd.to_numeric(data[col], errors='coerce')
    if(data[col].isnull().sum() >= 200):
        data.drop(col, 1, inplace=True)

Now that we have dropped sparse columns, we will want to impute the rest of the missing data. To do this, we use the current attributes mean and populate the missing data with that mean. This is the mathematically closest operation to do to find the expected value's of the missing entries. We can make use of Pandas' fillna function, which will take in a Series (a column of our data) and fill in all the NaN's with the mean of that column. The fillna function requires that missing data be in the format of NA/NaN values, so that is why we went through the "trouble" to convert missing entries from '?' to NaN objects.

Step 4: Analysis, Hypothesis testing & Machine Learning


Analysis and Machine Learning are very important in data science. Sometimes people need to predict values or classify things for many purposes. For this tutorial, we will show you how to predict violent crime rates for each community in our data based on the 100 remaining attributes, because communityname should not help with predictions and ViolentCrimesPerPop is what we are trying to predict. We had 1994 rows of data, we chose a random subset of 80% of the data for training data, and the remaining 20% was used to test our regression.


Regression

We choose to use two types of linear regression and two types of non-linear to see if the data was linear. The regressions we did were: Linear Regression, Stochastic Gradient Descent Regression, Extra Trees, and Random Forests. Linear regression uses ordinary least squares to fit a line to the data. Stochastic gradient descent mimimizes squared loss while adding a regularizing penalty function to promote sparsity in the weight vector. Specifically, we used the elastic net penalty function, which is a combination of an L1 and L2 loss function. For further reading on how this function works, please see this link. Extra Trees and Random Forests combine a set of randomized decision trees on various subsets of the data. They average these models to help with over-fitting and improve accuracy. Random Forests differ from Extra Trees in how they choose to split and sample the data given. Extra Trees tend to have more leaves, while Random Forests allow for sampling with replacement.


Goal

We want to build a model that uses the features to predict the ViolentCrimesPerPop: total number of violent crimes per 100K popuation data. The tree models we used (Extra Trees and Random Forests) also provide the "most important" features used to make these predictions.

Hypothesis

We believe that the most important features will be the percent of unemployed per community and the percent of illegal immigrants per community, due to their strong positive linear relationships.
